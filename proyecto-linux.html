Checklist de Seguridad

  • OS: Ubuntu Server 22.04 LTS

  • Brute Force: Fail2Ban Integration
  • Audit: Lynis Security Scan

Securización del Acceso SSH

La primera fase del hardening consistió en blindar el acceso remoto. Se deshabilitó el acceso al usuario root y se cambió el puerto por defecto para mitigar ataques de bots automatizados.

# Editando configuración de SSH
sudo nano /etc/ssh/sshd_config

# Cambios aplicados:
Port 2222
PermitRootLogin no
PasswordAuthentication no
AllowUsers orioladmin

Control de Tráfico con UFW

Se implementó una política de denegación por defecto. Solo se abrieron los puertos estrictamente necesarios para los servicios en ejecución, aplicando el principio de mínimo privilegio.

# Configuración de reglas básicas
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 2222/tcp
sudo ufw enable

Prevención de Intrusiones (Fail2Ban)

Para proteger el sistema contra ataques de fuerza bruta, se configuró Fail2Ban. El sistema monitoriza los logs de autenticación y banea automáticamente las direcciones IP que muestran un comportamiento sospechoso tras 3 intentos fallidos.

# Estado del baneo para SSH
sudo fail2ban-client status sshd
|- Filter
|  |- Currently failed: 2
|  `- Total failed: 15
`- Actions
   |- Currently banned: 1
   `- Total banned: 5
